### Task 1: audit_log table + LogAudit + wire CRUD

**Files:** `internal/db/schema.go`, `internal/core/audit.go` (new), `internal/db/repository.go`, test `internal/db/audit_test.go` (new), `internal/core/audit_test.go` (new, optional for the diff helper).

**Interfaces:**
- `core.AuditEntry{TS string; Aktion string; Entitaet string; Schluessel string; Details string}` + `core.DiffFields(old, new CSVRow) string` (JSON of changed fields old→new for a documented field set).
- `db.(*Repository).LogAudit(e core.AuditEntry) error`; `db.(*Repository).AuditLog(limit int) ([]core.AuditEntry, error)`.

- [ ] **Step 1:** Add to `schemaSQL`:
```sql
CREATE TABLE IF NOT EXISTS audit_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    ts DATETIME DEFAULT CURRENT_TIMESTAMP,
    aktion TEXT NOT NULL,
    entitaet TEXT,
    schluessel TEXT,
    details TEXT
);
CREATE INDEX IF NOT EXISTS idx_audit_ts ON audit_log(ts);
```

- [ ] **Step 2: core test** (`internal/core/audit_test.go`): `DiffFields` on two CSVRows differing in Gegenkonto + Bruttobetrag returns a JSON string containing both field names and old→new values; identical rows → `"{}"` or empty. Implement `core.AuditEntry` + `DiffFields(old,new CSVRow) string` in `internal/core/audit.go` (compare a fixed list: Auftraggeber, Rechnungsnummer, Rechnungsdatum, BetragNetto, SteuersatzBetrag, Bruttobetrag, Gegenkonto, Bankkonto, Bezahldatum, BuchungRef, Belegnummer, Ausgangsrechnung; emit a `map[string]struct{Alt,Neu any}` marshaled to JSON, only changed keys). Run → pass.
- [ ] **Step 3: db test** (`internal/db/audit_test.go`, mirror the temp-DB setup): after `Insert`, `AuditLog(10)` contains a `create` entry for that dateiname; after `Update`, an `update` entry whose details mention a changed field; after `Delete`, a `delete` entry. Run → fail.
- [ ] **Step 4:** Implement `LogAudit` (INSERT into audit_log) + `AuditLog(limit)` (SELECT ... ORDER BY ts DESC, id DESC LIMIT ?). Wire:
  - `Insert`: after success, `LogAudit({Aktion:"create", Entitaet:"invoice", Schluessel: row.Belegnummer+" "+row.Dateiname, Details: ""})` (best-effort; log a warning on audit error, don't fail the op).
  - `Update`: BEFORE updating, fetch the existing row via a helper (`getByKey(jahr,monat,oldDateiname)` — reuse `List`+filter or a small SELECT); compute `core.DiffFields(old,row)`; after success `LogAudit({Aktion:"update", ..., Details: diff})`.
  - `Delete`: before deleting, optionally fetch for the key; after success `LogAudit({Aktion:"delete", Schluessel: dateiname})`.
- [ ] **Step 5:** run → pass + full db & core. Commit `E20.1: audit_log table + LogAudit wired into CRUD + DiffFields`.
