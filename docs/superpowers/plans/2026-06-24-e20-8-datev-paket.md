### Task 1: core export-package builder

**Files:** `internal/core/exportpackage.go` (new), `internal/core/exportpackage_test.go` (new).

**Interface:**
```go
type BelegFile struct { Belegnummer, Dateiname string; Bytes []byte }
// BuildExportPackage builds a ZIP: "DATEV-EXTF_<period>.csv" (datevCSV), "belege/<sanitized belegnr or dateiname>.pdf" per BelegFile,
// "manifest.csv" (Belegnummer;Dateiname;Auftraggeber;Rechnungsdatum;Bruttobetrag;Gegenkonto), and "index.xml" (GoBD/GDPdU-style description referencing manifest.csv + the DATEV file).
func BuildExportPackage(rows []CSVRow, datevCSV []byte, belege []BelegFile, period string) ([]byte, error)
```

- [ ] **Step 1: test**: rows=2, datevCSV=[]byte("EXTF;..."), belege=2 fake PDFs. `BuildExportPackage` returns a non-empty zip; open it with `zip.NewReader` and assert it contains `DATEV-EXTF_<period>.csv`, `manifest.csv`, `index.xml`, and `belege/<...>.pdf` for each beleg; manifest has a header + 2 rows; index.xml is well-formed (`xml.Unmarshal` of a probe). Run → fail.
- [ ] **Step 2:** implement with `archive/zip`. manifest.csv: `;`-separated, UTF-8. index.xml: a GoBD-oriented description (a `<DataSet>` with `<Media>` → `<Table>` entries for `manifest.csv` and the DATEV file, listing columns — pragmatic, well-formed XML; document it as "GoBD-orientiert, nicht DTD-zertifiziert"). Sanitize beleg filenames (reuse `core` sanitize if present, else inline). Run → pass + full core.
- [ ] **Step 3:** Commit `E20.8: BuildExportPackage (DATEV + Belege + manifest + GoBD index.xml)`.
