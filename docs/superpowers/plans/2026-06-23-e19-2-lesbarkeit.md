# E19.2 — Lesbarkeit — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** Make the invoice table readable at a glance: account numbers carry their name, every status/account cell has an always-on Klartext tooltip enriched with more data, and a legend explains the symbols.

**Architecture:** The table renders string cells via a custom `hoverLabel` (`internal/ui/table.go`) that today shows a tooltip ONLY when the text is truncated. We add an `alwaysTooltip` flag so designated cells (account, status) show an enriched tooltip on hover regardless; the Gegenkonto cell gains the account name; status symbols are unified and explained by a legend popover. NOTE: Fyne string-table cells can't easily colour individual cell text, so this phase uses consistent SYMBOLS + tooltips + a legend (not full-colour icons) — an honest fit to the framework.

**Tech Stack:** Go 1.25, Fyne. `internal/core`, `internal/ui`. Branch `feat/e19-2-lesbarkeit`.

```go
package core

import "fmt"

// accountTypeWord maps an SKR account type to a German label for tooltips.
var accountTypeWord = map[string]string{
	"expense": "Aufwand", "revenue": "Erlös", "asset": "Aktiva",
	"liability": "Passiva", "equity": "Eigenkapital",
}

// AccountDisplay renders an account as "Nummer — Name" for compact cells.
func AccountDisplay(a SKRAccount) string {
	return fmt.Sprintf("%d — %s", a.Number, a.Name)
}

// AccountTooltip renders "Nummer — Name (Typ)" for hover, omitting the type
// when unknown.
func AccountTooltip(a SKRAccount) string {
	if w, ok := accountTypeWord[a.Type]; ok {
		return fmt.Sprintf("%d — %s (%s)", a.Number, a.Name, w)
	}
	return fmt.Sprintf("%d — %s", a.Number, a.Name)
}
```
