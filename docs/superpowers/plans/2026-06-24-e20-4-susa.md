**Tech:** Go 1.25, Fyne. `internal/core`, `internal/ui`. Branch `feat/e20-4-susa`.

### Task 1: core SuSa + GuV + PDFs

**Files:** `internal/core/susa.go` (new), `internal/core/susa_test.go` (new), `internal/core/pdfreport.go` (append two builders).

**Interfaces:**
```go
type AccountBalance struct { Konto int; Name string; SollSumme, HabenSumme, Saldo float64 }
func ComputeSuSa(rows []CSVRow, chart *ChartOfAccounts) []AccountBalance   // one row per konto, sorted by konto; Saldo=Soll-Haben; Name via chart
type GuV struct { ErloesPosten, AufwandPosten []AccountBalance; ErloeseGesamt, AufwandGesamt, Ergebnis float64 }
func ComputeGuV(susa []AccountBalance, chart *ChartOfAccounts) GuV          // partition by chart Type: revenue→Erlös (use HabenSumme-SollSumme), expense→Aufwand (SollSumme-HabenSumme); Ergebnis=Erlöse-Aufwand
```

- [ ] **Step 1: test**: two rows with bookings — e.g. an expense booking (Soll 4663 100 / Haben 1200 100) and a revenue booking (Soll 1200 200 / Haben 8400 200). Assert `ComputeSuSa`: konto 4663 Soll 100/Saldo 100; 8400 Haben 200/Saldo −200; 1200 Soll 200 Haben 100 Saldo 100. `ComputeGuV` (give chart with 4663=expense, 8400=revenue, 1200=asset): ErloeseGesamt 200, AufwandGesamt 100, Ergebnis 100. Run → fail.
- [ ] **Step 2:** implement susa.go. Iterate every `row.Buchung.Entries`, accumulate Soll/Haben per konto into a map, build sorted slice, resolve Name. Run → pass.
- [ ] **Step 3:** `BuildSuSaPDF(bals []AccountBalance, title string)` (cols Konto · Name · Soll · Haben · Saldo + totals) and `BuildGuVPDF(g GuV, title string)` (Erlöse section, Aufwand section, Ergebnis) in pdfreport.go; tiny PDF tests (%PDF). Run → pass + full core.
- [ ] **Step 4:** Commit `E20.4: ComputeSuSa + ComputeGuV + PDFs`.
